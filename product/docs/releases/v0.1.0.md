# v0.1.0 — Foundation

**Date:** 2026-07-16
**Research pins:** Observation Language `v1`, profile `interface.v1`, methodology laws (current).

The first release of Fingerprint the product: a conversational Skill that acts as a reflection
partner for people designing and building interfaces. It observes what is concretely present,
asks about intent, and hands automatic decisions back to the person as choices. It never
generates, templates, prescribes, judges, or decides.

## What a user would feel

A calm, perceptive design director in conversation. Shares an interface (URL, screenshot,
code, or description) and gets grounded questions that make the automatic decisions visible —
never instructions, never a verdict, never a redesign.

## What shipped

- The product pillar `product/`, cleanly separated from the research infrastructure.
- Product architecture, roadmap, and versioning docs.
- The Skill artifact `product/fingerprint/`:
  - `SKILL.md` — identity, the four constitutional laws, when to engage, the core loop,
    decline behavior.
  - `references/voice.md`, `reflection-flows.md`, `observation-bridge.md`,
    `capabilities-and-limits.md`.
  - `examples/session-transcripts.md` — worked sessions and anti-patterns.
  - `VERSION` = 0.1.0.

## Known limits (targets for later versions)

- Assumes a single screen; no multi-screen/flow observation yet (v0.2).
- No cross-session memory yet (v0.3).
- Degraded-input robustness not yet hardened (v0.4).
